# Next.js CMS Page Scaffolding

Follow existing project conventions to complete a full CMS CRUD list page in 9 steps.

## File Structure Convention

```
src/
├── graphql/
│   ├── fragments/{domain}/
│   │   └── {entity}-fields.fragment.graphql
│   ├── queries/{domain}/
│   │   └── get-{entities}.query.graphql
│   ├── mutations/{domain}/
│   │   ├── create-{entity}.mutation.graphql
│   │   ├── update-{entity}.mutation.graphql
│   │   └── delete-{entity}.mutation.graphql
│   └── generated/
│       └── graphql.ts              ← auto-generated by codegen
├── app/cms/{domain}/
│   ├── page.tsx                    ← Main page component
│   ├── page.module.scss            ← Page styles
│   └── _components/
│       ├── {Entity}Table/
│       │   ├── {Entity}Table.tsx
│       │   ├── {entity}-table.module.scss
│       │   └── index.ts
│       ├── {Entity}FormModal/
│       │   ├── {Entity}FormModal.tsx
│       │   ├── {entity}-form-modal.module.scss
│       │   └── index.ts
│       └── Delete{Entity}Dialog/   ← Optional
│           ├── Delete{Entity}Dialog.tsx
│           ├── delete-{entity}-dialog.module.scss
│           └── index.ts
```

## 9-Step Creation Workflow

### Step 1: Create Fragment

Create `src/graphql/fragments/{domain}/{entity}-fields.fragment.graphql`:

- Define fields needed for list/detail views
- Naming convention: `{entity}-{scope}-fields.fragment.graphql` (e.g., `category-list-fields.fragment.graphql`)

### Step 2: Create Query

Create `src/graphql/queries/{domain}/get-{entities}.query.graphql`:

- Fetch list data (with pagination params `offset`, `limit`)
- Reference the fragment created in Step 1
- Naming convention: `get-{entities}.query.graphql` (e.g., `get-backstage-categories.query.graphql`)

### Step 3: Create Mutations

Create mutation files under `src/graphql/mutations/{domain}/`:

- `create-{entity}.mutation.graphql`
- `update-{entity}.mutation.graphql`
- `delete-{entity}.mutation.graphql`
- Naming convention: `{action}-{entity}.mutation.graphql` (e.g., `create-category.mutation.graphql`)

### Step 4: Run Codegen

```bash
pnpm codegen
```

Outputs `src/graphql/generated/graphql.ts`, auto-generating Apollo hooks (`useGet{Entities}Query`, `useCreate{Entity}Mutation`, etc.).

Codegen config (`codegen.ts`):
- Scans `src/graphql/fragments/**/*.fragment.graphql`
- Scans `src/graphql/queries/**/*.query.graphql`
- Scans `src/graphql/mutations/**/*.mutation.graphql`
- Uses `typescript` + `typescript-operations` + `typescript-react-apollo` plugins
- `withHooks: true`

### Step 5: Page Component

Create `page.tsx` containing:
- `'use client'` directive
- Pagination state (`page`, `PAGE_SIZE`)
- Filter state (if needed)
- Modal/Dialog toggle state
- Apollo query/mutation hooks
- `useCallback` wrapping all handlers
- JSX structure: `PageHeader` → Filter (optional) → Table → FormModal → DeleteDialog

See [PAGE_TEMPLATE.md](./PAGE_TEMPLATE.md)

### Step 6: SCSS Module

Create `page.module.scss` using grid layout:

```scss
.page {
  display: grid;
  grid-template-rows: auto 1fr;  // or auto auto 1fr (when filter exists)
  row-gap: 16px;
  width: 100%;
  height: 100%;
}

.content {
  padding: 0 var(--mzn-spacing-padding-horizontal-spacious);
}
```

### Step 7: Table Component

Create `{Entity}Table` containing:
- `TableColumn` definitions (`useMemo`)
- `dataSource` transformation (`useMemo`)
- `actions` column (edit, delete buttons)
- Pagination config (`pagination` prop)
- Empty state config (`emptyProps`)

See [PAGE_TEMPLATE.md](./PAGE_TEMPLATE.md)

### Step 8: Form Modal

Create `{Entity}FormModal` containing:
- `react-hook-form` + `yup` schema
- Mezzanine `Modal` + `ModalHeader` + `ModalFooter`
- `FormField` + `Input` bindings
- Create/edit dual mode (determined by whether `entity` prop is null)

See [FORM_MODAL_TEMPLATE.md](./FORM_MODAL_TEMPLATE.md)

### Step 9: Integration & Verification

- Verify all Apollo hooks are correctly imported
- Verify `Message.success` / `Message.error` toast messages
- Verify `refetch()` is called after successful mutations
- Verify `handleClose` properly resets form state
