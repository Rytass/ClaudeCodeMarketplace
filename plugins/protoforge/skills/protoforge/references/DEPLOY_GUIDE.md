# Deployment Guide (Optional)

Deployment is **optional** — prototypes work as standalone local Next.js projects. When the user requests deployment, follow these steps.

## Prerequisites

- **GitHub CLI** (`gh`): For pushing to GitHub organization
- **Wrangler CLI** (`npx wrangler`): For Cloudflare Pages deployment
- Both require prior authentication (`gh auth login`, `wrangler login`)

## Generated Project Configuration

The prototype must use static export for Cloudflare Pages compatibility:

```js
// next.config.js
/** @type {import('next').NextConfig} */
const nextConfig = {
  output: 'export',
  images: {
    unoptimized: true,
  },
};

module.exports = nextConfig;
```

## Option A: Push to GitHub Organization

```bash
# Initialize git repo
cd {projectDir}
git init
git add .
git commit -m "feat: initial prototype generated by ProtoForge"

# Create repo in organization and push
gh repo create {orgName}/{projectName} --private --source=. --push
```

**Result**: `https://github.com/{orgName}/{projectName}`

Team members can then:
```bash
git clone https://github.com/{orgName}/{projectName}.git
cd {projectName}
npm install
npm run dev
```

## Option B: Deploy to Cloudflare Pages

```bash
# Build the project
cd {projectDir}
npm install
npm run build
# Static output lands in `out/` directory

# Create Pages project (first time only)
npx wrangler pages project create {projectName}

# Deploy
npx wrangler pages deploy out/ --project-name={projectName}
```

**Result**: `https://{projectName}.pages.dev`

## Option C: Both

Run GitHub push first, then Cloudflare Pages deploy. Order matters because `gh repo create --source=. --push` initializes the git repo.

## Deployment Checklist

Before deploying, verify:

1. `npm run build` succeeds without errors
2. `out/` directory exists and contains HTML files
3. No environment variables are required (all data is mock)
4. No API calls in the code (all mock data via useState hooks)

## Troubleshooting

### GitHub CLI (`gh`)

| Symptom | Cause | Fix |
|---------|-------|-----|
| `gh auth login` hangs | Firewall/VPN blocking OAuth callback | Use `gh auth login --with-token` with a PAT instead |
| `gh repo create` fails with 403 | Token lacks `repo` scope | Re-auth: `gh auth login -s repo` |
| `gh repo create` fails with 422 | Repo name already exists in org | Choose a different `--name` or delete the existing repo first |

### Wrangler CLI

| Symptom | Cause | Fix |
|---------|-------|-----|
| `wrangler login` browser popup not appearing | Headless/SSH environment | Use `CLOUDFLARE_API_TOKEN` env var instead of interactive login |
| `wrangler pages deploy` fails with 401 | Token expired or wrong account | Re-auth: `npx wrangler login` |
| `wrangler pages project create` fails | Project name contains invalid chars | Use kebab-case, lowercase only, no special chars |

### Build Issues

| Symptom | Cause | Fix |
|---------|-------|-----|
| `Module not found: @mezzanine-ui/*` | `npm install` incomplete | Delete `node_modules` and re-run `npm install` |
| Build fails with Node version error | Node < 18 | Upgrade Node to 18+ (`nvm use 18`) |
| `out/` directory empty after build | Missing `output: 'export'` in config | Verify `next.config.js` has `output: 'export'` |
| Cloudflare Pages 404 on subpages | Missing trailing-slash config | Add `trailingSlash: true` to `next.config.js` |
