# Deploy Prototype

Deploy a generated Next.js prototype to GitHub and/or Cloudflare Pages. This is an **optional** step — prototypes work as standalone local projects.

## Prerequisites

Check that required tools are installed **and authenticated** before attempting deployment:

- **GitHub**: `gh --version` — install via `brew install gh`, then `gh auth login`
  - **Auth check**: Run `gh auth status` — must exit 0. If not authenticated, tell user to run `gh auth login` and stop
- **Cloudflare Pages**: `npx wrangler --version` — authenticate via `npx wrangler login`
  - **Auth check**: Run `npx wrangler whoami` — must show authenticated user. If not, tell user to run `npx wrangler login` and stop

**CRITICAL**: Always verify authentication before any deploy action. Do NOT proceed with partial deployment if auth fails.

## Deployment Options

### GitHub Push

Pushes the entire source code to a private repo in the user's GitHub organization. Team members can clone and continue development.

```bash
cd {projectDir}
git init
git add .
git commit -m "feat: initial prototype generated by ProtoForge"
gh repo create {orgName}/{projectName} --{visibility} --source=. --push
# visibility: --private (default) or --public (if user requests)
```

### Cloudflare Pages

Deploys the static build output for instant preview. Requires `output: 'export'` in next.config.js.

```bash
cd {projectDir}
npm run build
npx wrangler pages project create {projectName} 2>/dev/null || true
npx wrangler pages deploy out/ --project-name={projectName}
```

### Both

Run GitHub push first (initializes git), then Cloudflare Pages deploy.

## Verification

After deployment, verify:
- **GitHub**: `gh repo view {orgName}/{projectName}` shows the repo
- **Cloudflare**: The deployment URL returns HTTP 200
